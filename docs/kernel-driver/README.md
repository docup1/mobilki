# Kernel Driver

Модуль ядра Linux, системный поток читает котировки из внешнего источника.

## Задачи

- Запускаться как kernel thread при загрузке модуля
- Непрерывно читать котировки из внешнего источника
- Сохранять котировки в ring buffer в памяти ядра
- Предоставлять character device `/dev/quotes` для чтения из user space
- Будить ожидающие процессы при появлении новых данных

## Структура проекта

| Путь | Что внутри |
|------|-----------|
| `src/quotes_drv.c` | Основной файл драйвера |
| `src/quotes_drv.h` | Заголовочный файл: структуры, константы |
| `src/thread.c` | Kernel thread — чтение котировок |
| `src/device.c` | Регистрация character device |
| `src/ringbuf.c` | Ring buffer в памяти ядра |
| `src/protocol.c` | Парсинг данных от внешнего источника |
| `Makefile` | Сборка kernel module |
| `Kconfig` | Конфигурация для menuconfig ядра |

## Компоненты

| Компонент | Файл | Что делает |
|-----------|------|-----------|
| Kernel thread | `thread.c` | Системный поток, непрерывный опрос источника |
| Character device | `device.c` | `/dev/quotes`, интерфейс для user space |
| Ring buffer | `ringbuf.c` | Кольцевой буфер, хранит последние котировки |
| Waitqueue | `quotes_drv.c` | Будит процессы при новых данных |
| Protocol parser | `protocol.c` | Разбирает сырые данные от источника |

## Структура модуля

| Функция | Что делает |
|---------|-----------|
| `quotes_init()` | Инициализация модуля: регистрация device, создание thread |
| `quotes_exit()` | Выгрузка модуля: остановка thread, unregister device |
| `quotes_thread()` | Главный цикл kernel thread |
| `quotes_open()` | Открытие device процессом из user space |
| `quotes_read()` | Чтение котировки из ring buffer |
| `quotes_poll()` | Ожидание данных через poll |
| `quotes_release()` | Закрытие device |

## Kernel thread

Главный цикл thread работает непрерывно.

| Шаг | Что происходит |
|-----|---------------|
| 1 | Опрос внешнего источника котировок |
| 2 | Парсинг сырых данных через protocol parser |
| 3 | Запись котировки в ring buffer |
| 4 | Wake up всех ожидающих процессов через waitqueue |
| 5 | Проверка kthread_should_stop() — сигнал к остановке |
| 6 | Sleep на короткий интервал, возврат к шагу 1 |

Thread использует `kthread_run()` для создания и `kthread_stop()` для остановки при выгрузке модуля.

## Character device

| Параметр | Значение |
|----------|----------|
| Имя | `/dev/quotes` |
| Major number | Динамический, через `alloc_chrdev_region()` |
| Minor number | 0 |
| Режим доступа | Read-only для всех |

Device file operations:

| Операция | Функция | Что делает |
|----------|---------|-----------|
| open | `quotes_open()` | Проверяет, что модуль загружен |
| read | `quotes_read()` | Копирует котировку из ring buffer в user space через `copy_to_user()` |
| poll | `quotes_poll()` | Добавляет процесс в waitqueue, возвращает POLLIN при наличии данных |
| release | `quotes_release()` | Убирает процесс из списка подписчиков |

## Ring buffer

Кольцевой буфер в памяти ядра. Хранит последние N котировок.

| Параметр | Значение |
|----------|----------|
| Размер | 4096 записей |
| Аллокация | `vmalloc()` при инициализации |
| Синхронизация | Spinlock для защиты от конкурентного доступа |
| Переполнение | Старые данные перезаписываются |

| Операция | Функция | Что делает |
|----------|---------|-----------|
| push | `ringbuf_push()` | Записывает котировку, сдвигает write index |
| pop | `ringbuf_pop()` | Читает котировку, сдвигает read index |
| empty | `ringbuf_empty()` | Проверяет, есть ли данные |
| size | `ringbuf_size()` | Количество доступных записей |

## Формат данных

Котировка в ring buffer — бинарная структура.

| Поле | Тип | Описание |
|------|-----|----------|
| ticker_len | u8 | Длина тикера |
| ticker | char[] | Тикер инструмента |
| price | u64 | Цена (в минимальных единицах, например центах) |
| volume | u64 | Объём |
| timestamp | u64 | Unix timestamp в наносекундах |
| bid | u64 | Цена покупки |
| ask | u64 | Цена продажи |
| checksum | u32 | Контрольная сумма |

## Внешний источник

Драйвер подключается к внешнему источнику котировок. Тип источника определяется конфигурацией.

| Тип источника | Механизм подключения |
|---------------|---------------------|
| Сетевой шлюз | Raw socket, чтение из сетевого интерфейса |
| PCIe устройство | MMIO, чтение из памяти устройства |
| Serial port | Чтение из последовательного порта |

Конкретный механизм чтения вынесен в отдельный блок, который можно заменить без изменения остального драйвера.

## Установка

| Команда | Что делает |
|---------|-----------|
| `make` | Сборка kernel module |
| `sudo insmod quotes_drv.ko` | Загрузка модуля в ядро |
| `lsmod \| grep quotes` | Проверка загрузки |
| `ls -la /dev/quotes` | Проверка создания device |
| `sudo rmmod quotes_drv` | Выгрузка модуля |

## Логирование

Драйвер использует `pr_info()`, `pr_err()`, `pr_debug()` для логирования.

| Уровень | Когда |
|---------|-------|
| pr_info | Загрузка/выгрузка модуля, подключение к источнику |
| pr_err | Ошибка чтения источника, ошибка аллокации |
| pr_debug | Каждая полученная котировка (отключено по умолчанию) |

Логи доступны через `dmesg` и `journalctl -k`.
