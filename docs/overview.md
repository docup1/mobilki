# Обзор системы

## Компоненты

| Компонент | Что делает | Технология |
|-----------|-----------|-----------|
| Kernel Driver | Модуль ядра Linux, системный поток читает котировки из внешнего источника | C, kernel space |
| Quote Service | Читает котировки из kernel driver, отдаёт по gRPC, пишет в Redis | Go, user space |
| Gateway | Превращает gRPC в REST, кэширует ответы, проверяет JWT | Go |
| Mobile App | Клиентское приложение для просмотра котировок и торговли | React Native |

## Как общаются

Kernel Driver — модуль ядра Linux. Работает как системный поток (kernel thread), непрерывно читает котировки из внешнего источника и пишет их в character device `/dev/quotes`.

Quote Service открывает `/dev/quotes` и читает котировки через системные вызовы `read()` и `poll()`. Это стандартное взаимодействие user space ↔ kernel space.

Quote Service отдаёт котировки по gRPC — это внутренний протокол, наружу он не выходит. Gateway подписывается на gRPC-стрим от Quote Service и одновременно принимает REST-запросы от мобильного приложения.

Mobile App общается только с Gateway и только по REST.

## Kernel Driver

Драйвер — loadable kernel module (.ko). При загрузке создаёт character device `/dev/quotes` и запускает kernel thread.

| Компонент драйвера | Что делает |
|--------------------|-----------|
| Kernel thread | Системный поток, читает котировки из внешнего источника |
| Character device | `/dev/quotes`, интерфейс для user space |
| Ring buffer | Кольцевой буфер в памяти ядра, хранит последние котировки |
| Poll/waitqueue | Будит Quote Service при появлении новых данных |

Подробнее: [Kernel Driver](../kernel-driver/README.md)

## Real-time через polling

Мобильное приложение опрашивает Gateway каждую секунду. Gateway кэширует котировки на 1 секунду — это значит, что даже при ста запросах в секунду от разных клиентов, в Quote Service уйдёт только один запрос. Остальные получат ответ из кэша.

Когда экран не в фокусе — polling останавливается. Это экономит батарею и трафик.

## Почему так

| Решение | Почему |
|---------|--------|
| Kernel driver | Котировки читаются на уровне ядра, минимальная задержка, изоляция от user space |
| Character device | Стандартный механизм ядра, `read()` и `poll()` работают из любого языка |
| REST вместо WebSocket | Проще, надёжнее, не ломается при NAT и firewall |
| Кэш в Gateway | Гасит нагрузку от polling, Quote Service не страдает |
| gRPC внутри | Типизированный контракт, streaming, быстро |
| DDD в Quote Service | Чёткая доменная модель, легко тестировать |
| FSD в мобилке | Явные зависимости, команда не мешает друг другу |
